# Unit Converter Sprint – Individual Reflection (Paul)

## 1. Project Scope

The main goal of this sprint is to create a **Unit Converter web application** that is efficient and well-presented. The application should be made to easily convert units of length, weight, and temperature. The ease of use and precision are given importance, and focus is placed on simplicity and effectiveness in user experience.

**Included Features:**
- A responsive, user-friendly input and unit selection interface.
- Strong algorithms for conversion in each category supported.
- Legible result displays, in addition to reset input and unit swapping options.
- Input validation controls to restrict invalid or blank submissions.
- Design compatibility on desktop and mobile platforms.

**Excluded Features:**
- Currency conversion and scientific units of higher order are outside the scope of the present project.
- Saving user history or preferences as a feature is postponed to future development stages.

---

## 2. Milestones and Timeline

### Initial Phase: "Game Structure with Copilot"
The first step is to plan the data structures and the unit conversion logic. In parallel, the basic layout will be created with the use of CSS to create the structure of the interface.

### Intermediate Phase: "Mid-Sprint Demo + Feedback"
By halfway, the functional prototype for at least two categories of units (length and temperature) should be functional. The user interface can still be basic, but it will be a foundation for demonstration and team feedback.

### Final Phase: Sprint Conclusion
By the end of sprint, the application should be ready and complete with all the targeted categories and a polished design. Any problems or ideas picked up in mid-sprint review will have been solved.

**Checkpoints:**
- Daily WhatsApp feedback to track progress and overcome barriers.
- Code review sessions prior to finalization.
- Scheduled time for rigorous testing and polishing before submission.

## 4. Expected Challenges and Solutions

**Possible Challenges:**  
- Ensuring better display and usability on smaller screens.  
- Seamless integration of design and functional logic.  
- Handling unusual input values and corner cases.
- Enforcing good input validation without overly complicating the interface.

**Mitigation Strategies:**
- Keeping responsive design as a top priority and performing cross-device testing.
- Defining plain interface-logic protocols to facilitate seamless integration.
- Adding input validation to avoid typical mistakes.
- Providing ample time for thorough testing and resolving problems before the final presentation.
